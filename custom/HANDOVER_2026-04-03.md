## Custom modules in this directory

Three modules implement a REST API for a decision-support workflow:

| Module | Entities | REST resources |
|---|---|---|
| `process/` | Process | 7 resources (create, get, list, update, step CRUD, delete/archive) |
| `decision_support/` | DecisionSupport | 7 resources (create, get, list, update, archive, duplicate, steps) |
| `decision_support_file/` | DecisionSupportFile | 3 resources (create, list, archive) |

These modules were reviewed and fixed in the old D10 project
(branch `chore/fix-custom-module-drupal-11-upgrade-blockers`) and then
copied into this D11 clean install. The fixes included:

- Archive/delete semantics resolved: `archiveDecisionSupport()` converted from
  hard-delete (`->delete()`) to soft-archive (consistent with process + file modules)
- All 15 REST endpoints assigned specific permissions (not generic 'access content')
- File access check added in DecisionSupportFileService
- N+1 query removed from decision support step loading
- Entity list queries switched to entity query + accessCheck(TRUE)
- Deprecated `text_processing` field settings removed
- Interface docblocks corrected
- Unused KeyValueStore injections removed
- Static Entity::load() calls replaced with entityTypeManager
- `create-consumer.php` OAuth script deleted (no longer used)
- PHPCS (Drupal + DrupalPractice) and PHPStan level 2 were both passing clean
  at time of copy

**Assumption:** the copied code is the post-fix version. This has not yet been
re-validated by running PHPCS/PHPStan on this D11 project directly.

### Not yet done — installation steps outstanding

Read `notes/User_Manual.txt` in this directory for full setup steps. Key items:

1. **Enable modules** — run drush command:
   ```
   vendor/bin/drush en consumers simple_oauth decision_support decision_support_file process jsonapi jsonapi_extras restui rest serialization -y
   ```
   Note: `investigation_builder`, `investigation_documents`, `report_builder` are
   referenced in the old manual — confirm whether these are still the correct module
   names or whether they have been renamed/replaced by the three custom modules above.

2. **Taxonomy** — add `status` vocabulary with terms: `draft`, `published`, `archived`

3. **OAuth setup** — the old manual references `create-consumer.php`. That script has
   been deleted and this approach is no longer used. The OAuth/authentication setup
   process needs to be documented as the install progresses. Check with owner on
   current approach (Simple OAuth module is still listed but consumer creation method
   has changed).

4. **OAuth keys** — generate public/private keys and place them **outside** the project
   root. Run `chmod 744` on both. Configure path at:
   `Configuration >> Simple OAuth`

5. **Private files path** — edit `web/sites/default/settings.php` and add:
   ```php
   $settings['file_private_path'] = '/path/outside/project/root/files';
   ```
   Then set Default Download Method to "Private local files served by Drupal" at:
   `Configuration >> Media >> File system`
   **This step is easy to forget and breaks file handling if missed.**

6. **Enable REST resources** — approximately 18 REST resources must be enabled
   manually at `Configuration >> REST`. See `notes/User_Manual.txt` for the full list.
   Verify the resource names match the new module names before enabling.

7. **Angular environment** — `DecisionSupport_angular` `environment.ts` still points
   to the old backend URL. Once the D11 site is running, add a new environment file
   targeting the new URL before any Angular integration testing.

8. **User_Manual.txt** — needs a full rewrite to reflect D11 clean install, current
   module names, removal of `create-consumer.php`, and correct paths. This is in scope
   for this coordination effort.
